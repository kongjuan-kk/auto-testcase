# 测试用例自动生成

将需求文档、产品描述或功能关键词转化为结构化的 Excel 测试用例文件（.xlsx）。

## 工作流程

### 第1步：获取输入

接受以下任意形式的输入：

- **纯文本**：用户直接粘贴的功能描述或需求片段
- **文件导入**：用户提供的 .txt / .md / .docx 需求文档
  - .txt / .md 文件：使用 Read 工具直接读取
  - .docx 文件：使用 `scripts/read_word.py` 脚本转换为文本
- **功能关键词**：如"用户登录功能""商品下单流程"

#### Word 文档读取方法

当用户提供 .docx 文件时，使用以下方法读取：

**方法1：使用 Python 脚本（推荐）**
```bash
python scripts/read_word.py "path/to/document.docx" > temp.txt
# 然后用 Read 工具读取 temp.txt
```

**方法2：使用 python-docx 库**
```bash
python -c "
from docx import Document
doc = Document('path/to/document.docx')
text = '\n'.join([para.text for para in doc.paragraphs])
print(text)
" > temp.txt
```

**方法3：命令行工具（如果可用）**
```bash
# 使用 pandoc（需安装）
pandoc -f docx -t plain "document.docx" -o temp.txt

# 使用 libreoffice（需安装）
soffice --headless --convert-to txt "document.docx"
```

如果输入过于简短（少于20字），主动询问用户补充以下信息：
- 所属模块名称
- 核心业务规则或约束
- 涉及的关键参数/字段

### 第2步：提取核心要素

从输入文本中识别并提取：

1. **模块名称**：功能所属的系统模块（如"用户中心""订单管理"）

   **通用识别规则**（按优先级）：
   - **显式标识**：文档中的明确标题、章节名称
   - **表格分类列**：当表格第一列包含分类标识（如：分类、类型、项目等），将同一分类下的内容归纳为一个模块
   - **功能区域归纳**：相邻且功能相关的多个功能点归纳为同一模块
   - **默认命名**：如无法识别，使用文档标题或"主功能模块"

2. **功能点**：具体功能动作，**必须细致拆分**：

   **通用识别规则**（按优先级）：
   - **表格行识别**：当内容在表格中时，每行（或相关行组）即为一个功能点
   - **联级关系识别**：当存在"A→B→C"或"联级/联动"描述时，归纳为联级功能点
   - **字段级别识别**：每个独立的输入字段、选择条件、操作按钮都是独立功能点
   - **功能类型识别**：筛选类、输入类、操作类、展示类

   **拆分原则**：
   - ✅ 正确做法：表格中每行、每个独立字段都是独立功能点
   - ✅ 正确做法：联级字段归纳为"联级筛选/选择"功能点
   - ❌ 错误做法：把多个字段合并为一个功能点
   - ❌ 错误做法：把表格整体作为一个功能点

3. **输入参数**：操作涉及的字段和参数（如手机号、密码、验证码）
4. **业务规则**：限制条件、校验规则（如"密码长度6-20位"、"字段不超过20个字符"）
5. **预期输出**：正常操作的结果（如"跳转首页""下单成功"）
6. **异常场景**：可能的错误情况（如空输入、格式错误、超时）

### 第3步：生成测试用例数据

对每个功能点，**必须**生成以下3类用例：

**正常场景（功能）**：合法输入、正确操作流程，验证基本功能正确性
**边界场景（边界）**：参数的最大值/最小值/临界值、空值、特殊字符
**异常场景（异常）**：错误输入、非法操作、网络异常、权限不足等

每个功能点至少生成5条用例（正常2条+边界1条+异常2条），复杂功能可适当增加。

---

#### 通用场景生成规则

**根据功能类型生成对应场景**：

**1. 输入类功能（文本框、数字框）**
- 正常：输入合法值、默认值
- 边界：字符限制（恰好达到、超过）、数值限制（最小、最大、0）
- 异常：空值、负数、特殊字符、格式错误

**2. 选择类功能（下拉框、单选、多选）**
- 正常：单选任意选项、多选多个选项
- 边界：选项数量极少（仅1个）、全选所有
- 异常：清空选择、选项为空或无数据

**3. 联级选择功能**
- 正常：逐级选择（A→B→C）、选择任意一级
- 边界：某级选项数量为0（无下级）、某级选项数量为1
- 异常：跳级选择、切换上级后下级处理（清空/更新/保持）

**4. 操作类功能（按钮）**
- 正常：正常点击操作
- 边界：连续/重复点击
- 异常：无权限操作、操作条件不满足、网络异常

**5. 筛选/过滤功能**
- 正常：单个筛选项过滤、多个筛选项组合过滤
- 边界：筛选结果为0条、筛选结果为全部数据
- 异常：筛选条件冲突、跳级筛选、数据加载失败、切换筛选后数据/统计更新

**6. 列表/排序功能**
- 正常：升序排序、降序排序
- 边界：列表为空、列表数据量极大
- 异常：排序字段值相同时处理、排序失败

**7. 日期/时间选择**
- 正常：选择今天、选择历史日期、选择日期范围
- 边界：选择最早/最晚可选日期、开始日期=结束日期
- 异常：开始日期晚于结束日期、选择未来日期（如不适用）

**8. 数据导出功能**
- 正常：导出全部数据、导出筛选后数据
- 边界：导出数据量为0、导出数据量极大（>5000条、>50万条）
- 异常：无权限导出、导出格式不支持、导出中断

---

**用例生成细化规则**：
- 每个下拉选择框：测单选、测多选、测默认值、测清空
- 每个文本输入框：测正常输入、测恰好达到字符限制、测超过字符限制、测特殊字符、测空值
- 每个按钮：测正常点击、测重复点击、测无权限点击
- 每个日期选择：测选择今天、测选择历史日期、测选择未来日期、测日期范围
- 每个筛选功能：测单选、多选、联级、跳级、清空、更新、数据加载失败

参考预设模板提升质量 -> 详见 [templates.md](templates.md)

### 第4步：组织用例数据

#### 核心字段（9个）

| 字段 | 说明 | 生成规则 |
|------|------|----------|
| 用例编号 | 唯一标识 | 格式：`MODxxx-FUNxxx-CASExxx`，自动递增 |
| 模块名称 | 所属模块 | 从输入提取，同一模块共享编号 |
| 功能点 | 具体功能 | 从输入提取核心功能动作 |
| 用例标题 | 场景描述 | 简洁概括测试场景，如"正确手机号+正确验证码登录成功" |
| 前置条件 | 执行前提 | 编号列表，如"1. 用户已注册；2. 网络正常" |
| 测试步骤 | 操作指引 | 编号列表，分步描述具体操作 |
| 预期结果 | 期望输出 | 与测试步骤对应的预期结果 |
| 优先级 | 高/中/低 | 正常场景=高，边界=中，异常=中/低 |
| 用例类型 | 分类标签 | 功能/边界/异常 |

#### 可扩展字段

用户可通过提示追加以下字段（按需添加到Excel末尾列）：
- 测试环境、缺陷编号、执行人、执行日期、实际结果、备注

如用户指定自定义字段，在Excel中追加对应列即可。

#### 编号规则

```
模块编号：MOD001, MOD002, ...（按模块出现顺序）
功能编号：FUN001, FUN002, ...（每个模块内从001开始）
用例编号：CASE001, CASE002, ...（每个功能内从001开始）
完整编号：MOD001-FUN001-CASE001
```

### 第5步：生成Excel文件

使用 `scripts/generate_xlsx.py` 脚本生成格式化的 .xlsx 文件。

#### 调用方式

先将用例数据组织为 JSON 格式，保存为临时文件，再调用脚本：

```python
# test_cases.json 结构
{
  "cases": [
    {
      "用例编号": "MOD001-FUN001-CASE001",
      "模块名称": "用户中心",
      "功能点": "手机号+验证码登录",
      "用例标题": "正确手机号+正确验证码登录成功",
      "前置条件": "1. 用户已注册\n2. 系统网络正常",
      "测试步骤": "1. 打开登录页面\n2. 输入已注册手机号\n3. 获取并输入正确验证码\n4. 点击登录按钮",
      "预期结果": "登录成功，跳转到首页，显示用户昵称",
      "优先级": "高",
      "用例类型": "功能"
    }
  ],
  "extra_columns": [],
  "title": "XXX功能测试用例"
}
```

```bash
python scripts/generate_xlsx.py test_cases.json output.xlsx
```

#### 若脚本执行失败

回退方案：使用 xlsx skill 生成Excel文件，确保：
- 表头行：深蓝背景(#1F4E79) + 白色加粗字体 + 居中
- 数据行：交替浅灰(#F2F2F2)和白色背景
- 列宽自适应内容长度（最小15字符，最大50字符）
- 所有单元格添加细边框
- 第一行冻结（冻结窗格）

### 第6步：输出文件

将生成的 .xlsx 文件保存到用户工作区，提供文件链接。

文件命名规则：`测试用例_{模块名称}_{YYYYMMDD}.xlsx`

## 质量标准

生成用例时遵守以下质量规则：

1. **用例标题**不超过30字，要能独立表达测试意图
2. **测试步骤**必须可执行，不写模糊表述（如"正确操作"）
3. **预期结果**必须可验证，包含具体值或状态描述
4. **前置条件**列出所有必要前提，不遗漏
5. **优先级**分配原则：核心流程=高，辅助流程=中，极端场景=低
6. 同一功能点的用例按 正常->边界->异常 顺序排列
7. 编号全局唯一，不允许重复

**高质量用例示例**：

❌ **错误示例**（功能点太粗）：
- 功能点：搜索功能
- 用例标题：输入门店编码搜索

✅ **正确示例**（功能点细致）：
- 功能点：搜索功能-门店编码
- 用例1：输入正确门店编码进行模糊搜索（功能）
- 用例2：门店编码恰好为20个字符进行搜索（边界）
- 用例3：门店编码超过20个字符提示超长（边界）
- 用例4：门店编码包含特殊字符进行搜索（异常）
- 用例5：门店编码为空点击查询（异常）

**覆盖度检查清单**：
- [ ] 每个输入字段都有独立的测试用例
- [ ] 每个字段的字符限制都有"恰好"和"超过"的边界测试
- [ ] 每个下拉框都有单选、多选、默认值测试
- [ ] 每个按钮都有正常点击、重复点击、无权限测试
- [ ] 异常场景覆盖：空值、超长、特殊字符、网络异常、权限不足

## 交互指引

- 如用户只给了关键词（如"登录功能"），先参考 templates.md 中的预设模板，再结合上下文生成
- 生成后告知用户用例总数、覆盖的模块/功能数量
- 主动提示用户是否需要：补充更多场景、调整优先级、增加自定义字段
