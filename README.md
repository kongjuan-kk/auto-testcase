# auto-testcase

> 🚀 自动生成标准化Excel测试用例的 Claude Code 技能

[![License](https://img.shields.io/badge/license-MIT-blue.svg)](LICENSE)
[![Claude Code](https://img.shields.io/badge/Claude-Code-purple.svg)](https://claude.com/claude-code)

## 📖 简介

**auto-testcase** 是一个强大的测试用例自动生成技能，能够从需求文档（.txt/.md/.docx）、功能描述或关键词快速生成结构化的 Excel 测试用例文件。

### 核心特性

- ✨ **多格式支持** - 支持 .txt、.md、.docx 需求文档导入
- 🎯 **智能提取** - 自动识别模块、功能点、业务规则
- 📊 **专业输出** - 生成格式精美的 Excel 测试用例（.xlsx）
- 🧩 **场景覆盖** - 自动生成正常、边界、异常三类测试场景
- 🎨 **样式精美** - 深蓝色表头、交替行色、优先级着色
- 📝 **模板丰富** - 内置常见业务场景测试用例模板

## 🚀 快速开始

### 环境要求

```bash
# Python 3.7+
pip install openpyxl python-docx
```

### 安装

将此项目放置到你的 Claude Code skills 目录：

```bash
~/.claude/skills/auto-testcase/
```

### 使用方式

在 Claude Code 中直接使用自然语言：

```
请为以下需求生成测试用例：
[粘贴你的需求文档或功能描述]
```

或者提供文档路径：

```
帮这个文档生成测试用例：C:/path/to/requirements.docx
```

## 📋 工作流程

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  输入需求    │ -> │  提取要素   │ -> │  生成用例   │ -> │  输出Excel  │
│  文档/描述  │    │  模块/功能  │    │  3类场景    │    │  .xlsx文件  │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
```

### 1️⃣ 获取输入

- **纯文本**：直接粘贴功能描述或需求片段
- **文件导入**：支持 .txt / .md / .docx 文档
- **功能关键词**：如"用户登录功能""商品下单流程"

### 2️⃣ 提取核心要素

自动识别并提取：
- 模块名称
- 功能点（细致拆分到字段级别）
- 输入参数
- 业务规则
- 预期输出
- 异常场景

### 3️⃣ 生成测试场景

为每个功能点自动生成：

| 场景类型 | 说明 | 用例数量 |
|---------|------|---------|
| **正常场景** | 合法输入、正确操作流程 | 2条 |
| **边界场景** | 最大值/最小值/临界值/空值 | 1条 |
| **异常场景** | 错误输入/非法操作/网络异常 | 2条 |

### 4️⃣ 输出 Excel

生成的 Excel 文件包含以下字段：

| 字段 | 说明 |
|------|------|
| 用例编号 | MODxxx-FUNxxx-CASExxx |
| 模块名称 | 所属功能模块 |
| 功能点 | 具体测试功能 |
| 用例标题 | 简洁场景描述 |
| 前置条件 | 执行前提条件 |
| 测试步骤 | 详细操作步骤 |
| 预期结果 | 期望输出 |
| 优先级 | 高/中/低 |
| 用例类型 | 功能/边界/异常 |

## 🎨 Excel 样式预览

生成的 Excel 文件具有以下特点：

- 🎨 **深蓝色表头**（#1F4E79）+ 白色加粗字体
- 📏 **交替行背景**（浅灰 #F2F2F2 / 白色）
- 🔢 **自动列宽**调整（最小15字符，最大50字符）
- 🔲 **所有单元格**添加细边框
- ❄️ **首行冻结**便于浏览
- 🏷️ **优先级着色**：高（红）/ 中（橙）/ 低（灰）
- 📊 **类型着色**：功能（蓝）/ 边界（黄）/ 异常（红）
- 🔍 **自动筛选**功能

## 📚 内置模板

项目内置了丰富的测试用例模板，涵盖常见业务场景：

1. **用户登录/注册** - 账号密码、手机验证码、记住密码
2. **商品下单/购买** - 购物车、提交订单、库存验证
3. **支付流程** - 微信支付、支付宝、银行卡
4. **表单提交** - 字段验证、格式校验、必填项
5. **搜索功能** - 关键词搜索、模糊匹配、结果排序
6. **文件上传/下载** - 格式验证、大小限制、断点续传
7. **权限管理** - 角色控制、菜单权限、操作权限
8. **筛选/过滤** - 单项筛选、组合筛选、联级筛选
9. **数据列表** - 分页、排序、导出
10. **配置管理** - 参数设置、默认值、边界值

详见 [templates.md](templates.md)

## 🔧 高级用法

### 自定义字段

可以在生成的 Excel 中追加额外字段：

```
生成测试用例，额外添加：测试环境、执行人、执行日期字段
```

### 使用 Python 脚本直接生成

```python
import json
import subprocess

# 准备测试用例数据
data = {
    "cases": [
        {
            "用例编号": "MOD001-FUN001-CASE001",
            "模块名称": "用户中心",
            "功能点": "手机号登录",
            "用例标题": "正确手机号+正确验证码登录成功",
            "前置条件": "1. 用户已注册\n2. 网络正常",
            "测试步骤": "1. 打开登录页\n2. 输入手机号\n3. 输入验证码\n4. 点击登录",
            "预期结果": "登录成功，跳转首页",
            "优先级": "高",
            "用例类型": "功能"
        }
    ],
    "extra_columns": ["测试环境", "执行人"],
    "title": "用户登录测试用例"
}

# 调用生成脚本
subprocess.run([
    'python',
    'scripts/generate_xlsx.py',
    json.dumps(data),
    'output.xlsx'
])
```

## 🎯 质量标准

生成的测试用例遵循以下质量原则：

- ✅ **用例标题**不超过30字，独立表达测试意图
- ✅ **测试步骤**可执行，不写模糊表述
- ✅ **预期结果**可验证，包含具体值或状态
- ✅ **前置条件**列出所有必要前提
- ✅ **优先级**合理分配：核心流程=高，辅助流程=中，极端场景=低
- ✅ 同一功能点的用例按 `正常->边界->异常` 顺序排列
- ✅ 编号全局唯一，不允许重复
